# Deep Learning-Driven SAR Automatic Target Recognition

Few-shot open-set recognition on the MSTAR dataset using a CNN+GNN pipeline, with optional GAN augmentation, physics-informed regularization, and rotation/speckle invariance training.

Reference paper: Zhou et al., "SAR Target Recognition with Limited Training Samples in Open Set Conditions", *Sensors* 2023.

---

## Setup

**Requirements**: Python 3.7+, PyTorch ≥ 1.8, CUDA-capable GPU (tested on NVIDIA RTX 3060 12 GB).

```bash
pip install -r requirements.txt
```

## Quick Start

### Paper-default run (3-way 20-shot, T72 unseen)

```bash
python main.py \
    --data_root mstar_sampled_data \
    --nway 3 \
    --shots 20 \
    --save
```

### 5-way configuration

```bash
python main.py \
    --data_root mstar_sampled_data \
    --nway 5 \
    --shots 20 \
    --save
```

### Resume from checkpoint

```bash
python main.py \
    --data_root mstar_sampled_data \
    --nway 3 \
    --shots 20 \
    --load \
    --load_dir model/3way_20shot_gnn_ \
    --save
```

## All Arguments

### Core

| Argument | Default | Description |
|----------|---------|-------------|
| `--data_root` | `data` | Path to the dataset root directory. Point this to `mstar_sampled_data` for the MSTAR dataset. |
| `--dataset` | `MSTAR` | Dataset name. Only `MSTAR` is currently supported. |
| `--model_type` | `gnn` | Model to use. `gnn` = few-shot GNN pipeline. `cnn` = closed-set CNN baseline. |
| `--nway` | `3` | Number of seen classes per episode (C-way). Paper evaluates C = 3, 5, 7. |
| `--shots` | `20` | Number of support images per class (K-shot). |
| `--seed` | `1` | Random seed for reproducibility across data sampling and weight init. |
| `--use_gpu` | `0` | GPU device index to use (passed to `CUDA_VISIBLE_DEVICES`). Set `""` to force CPU. |

### Training Loop

| Argument | Default | Description |
|----------|---------|-------------|
| `--todo` | `train` | Mode: `train` runs the training loop. `valid` is reserved for future use. |
| `--batch_size` | `8` | Number of episodes per training iteration. Each episode is one C-way K-shot task. |
| `--lr` | `0.001` | Adam optimizer learning rate. |
| `--max_iteration` | `100000` | Maximum number of training iterations before stopping regardless of early stop. |
| `--log_interval` | `100` | Print a training loss summary every N iterations. |
| `--eval_interval` | `500` | Run evaluation and check for a new best checkpoint every N iterations. |
| `--early_stop` | `10` | Stop training if overall accuracy does not improve for this many consecutive eval intervals. |

### Checkpointing

| Argument | Default | Description |
|----------|---------|-------------|
| `--save` | `False` | Save the best model checkpoint and training logs to disk. Checkpoint criterion is highest overall accuracy. |
| `--load` | `False` | Load a model checkpoint before training (or before `--eval_only`). |
| `--load_dir` | `model/3way_20shot_gnn_` | Directory containing the `model.pth` checkpoint to load. |
| `--model_root` | `model` | Root directory where model checkpoints are saved. |
| `--log_root` | `log` | Root directory where training logs are written. |
| `--affix` | `""` | Optional string appended to the auto-generated folder name (`{nway}way_{shots}shot_{model_type}_{affix}`). Useful to distinguish runs. |

### Open-Set Recognition

| Argument | Default | Description |
|----------|---------|-------------|
| `--unseen_class` | `T72` | Directory name of the class held out as "unseen" during training. Must exactly match a subdirectory name in `--data_root`. All images from this class go into the open-set test pool. |
| `--unseen_ratio` | `1.0` | Controls the proportion of unseen vs. seen queries sampled during training. `1.0` = 1:1 (50% unseen), `3.0` = 3:1 (75% unseen), `0.33` ≈ 1:3 (25% unseen). Paper Table 4 shows 1:1 achieves best overall accuracy. |

### GAN Augmentation

| Argument | Default | Description |
|----------|---------|-------------|
| `--gan_augment` | `False` | Augment each support set with synthetic images generated by the trained Conditional GAN. GAN must be trained first with `gan.py`. |
| `--gan_output_dir` | `gan_output` | Directory containing GAN-generated images, organized as `gan_output/<class_name>/`. |

Train the GAN first:

```bash
python gan.py \
    --data_root mstar_sampled_data \
    --output_dir gan_output \
    --epochs 200 \
    --n_generate 100
```

Then run with augmentation:

```bash
python main.py \
    --data_root mstar_sampled_data \
    --nway 3 \
    --shots 10 \
    --gan_augment \
    --gan_output_dir gan_output \
    --save
```

### Physics-Informed Regularization

| Argument | Default | Description |
|----------|---------|-------------|
| `--physics_lambda` | `0.0` | Weight λ for the intra-class embedding variance penalty added to the classification loss: `L_total = L_NLL + λ * L_physics`. Set to `0.0` to disable (pure classification). Typical value: `0.05`. |

The physics loss encodes the radar physics prior that same-class targets should produce consistent scattering signatures despite speckle noise variation.

```bash
# With physics regularization
python main.py \
    --data_root mstar_sampled_data \
    --nway 3 \
    --shots 20 \
    --physics_lambda 0.05 \
    --save

# Ablation: same config without regularization
python main.py \
    --data_root mstar_sampled_data \
    --nway 3 \
    --shots 20 \
    --physics_lambda 0.0 \
    --save
```

### Augmentation Transforms

| Argument | Default | Description |
|----------|---------|-------------|
| `--augment_rotation` | `False` | Apply a uniformly random rotation in [0°, 360°) to each support image at batch generation time. Trains invariance to azimuth angle variation. |
| `--augment_speckle` | `False` | Apply multiplicative Rayleigh-distributed speckle noise to each support image. Trains robustness to coherent imaging noise. |
| `--speckle_sigma` | `0.1` | Controls the intensity of speckle noise. Higher values = stronger noise. Typical range: 0.05–0.3. |

```bash
# With both augmentations
python main.py \
    --data_root mstar_sampled_data \
    --nway 3 \
    --shots 20 \
    --augment_rotation \
    --augment_speckle \
    --speckle_sigma 0.1 \
    --save
```

### Evaluation

| Argument | Default | Description |
|----------|---------|-------------|
| `--eval_only` | `False` | Skip training entirely. Load a checkpoint, run a single 5000-task evaluation, save a JSON report to `--eval_output`, then exit. Requires `--load`. |
| `--eval_output` | `results` | Directory where JSON evaluation reports are saved. Each report contains overall/seen/unseen accuracy, per-class precision/recall/F1, and a confusion matrix. |

```bash
# Evaluate a saved checkpoint without re-training
python main.py \
    --data_root mstar_sampled_data \
    --nway 3 \
    --shots 20 \
    --load \
    --load_dir model/3way_20shot_gnn_ \
    --eval_only \
    --eval_output results
```

### CNN Baseline

| Argument | Default | Description |
|----------|---------|-------------|
| `--baseline_kshot` | `False` | When using `--model_type cnn`, subsample exactly `--shots` images per class from the training split instead of using all available data. Used to measure CNN performance under the same data scarcity as the FSL setting. |

```bash
# Full-data CNN baseline (closed-set, 9 seen classes)
python main.py \
    --data_root mstar_sampled_data \
    --model_type cnn \
    --nway 9 \
    --save

# K=10 shot CNN baseline
python main.py \
    --data_root mstar_sampled_data \
    --model_type cnn \
    --nway 9 \
    --shots 10 \
    --baseline_kshot \
    --save
```

## Common Experiment Recipes

### Reproduce paper baseline (3-way 20-shot)

```bash
python main.py \
    --data_root mstar_sampled_data \
    --nway 3 \
    --shots 20 \
    --unseen_class T72 \
    --unseen_ratio 1.0 \
    --batch_size 8 \
    --lr 0.001 \
    --max_iteration 100000 \
    --early_stop 10 \
    --save
```

Expected: seen ≥ 95%, unseen ≥ 62%, overall ≥ 85%.

### Multi-way sweep (3 / 5 / 7-way)

```bash
for NWAY in 3 5 7; do
    python main.py \
        --data_root mstar_sampled_data \
        --nway $NWAY \
        --shots 20 \
        --affix sweep \
        --save
done
```

### GAN + physics + augmentation (full pipeline)

```bash
# Step 1: Train GAN
python gan.py \
    --data_root mstar_sampled_data \
    --output_dir gan_output \
    --epochs 200

# Step 2: Train FSL model with all enhancements
python main.py \
    --data_root mstar_sampled_data \
    --nway 3 \
    --shots 10 \
    --gan_augment \
    --physics_lambda 0.05 \
    --augment_rotation \
    --augment_speckle \
    --affix full_pipeline \
    --save
```

## Architecture Summary

**CNN Feature Extractor** (`cnn.py`): 4 × [Conv2d(32) + BN + ReLU + SE-attention + MaxPool] → Conv2d(64) global projection → 64-D embedding.

**GNN Relation Space** (`gnn.py`): 3 adjacency modules (pairwise |vᵢ − vⱼ| → 1×1 Conv MLP → softmax) alternating with 2 update modules (weighted aggregation → FC+ReLU), dense feature concatenation after each update. Output: (C+1) logits for the query node.

**Training objective**: NLL loss on C+1 classes (C seen + 1 unseen) + optional physics variance penalty.

**Evaluation**: 5000 episodic test tasks; reports seen accuracy, unseen accuracy, and overall accuracy separately.
